第十一天

方法的重写

子类拥有父类的方法和属性，子类继承自父类，自然就有了父类里定义的属性，可以直接使用父类里封装好的方法，不需要再次开发。

思考：当父类的方法不能满足子类需求时，子类怎样实现自己的功能？ 子类可以对父类的方法进行重写。

重写父类方法分为两种情况：

  1. 覆盖父类的方法，子类重新实现自己的功能
  2. 对父类方法的功能进行扩展。

子类覆盖父类方法

在开发中，如果父类的方法实现和子类的方法实现完全不同，就可以使用覆盖的方式，在子类中重新编写父类的方法实现。 相当于，在子类中又重新定义并实现了一个和父类重名的方法。根据类的__mro__属性，一个对象在调用方法时，首先会查看自己当前类里是否有这个方法，如果自己类里已经有了这个方法，就会直接调用，而不会找到父类里的方法。

class Father(object):
    def __init__(self):
        pass

    def eat(self):
        print('father在吃东西')

class Son(Father):
    def __init__(self):
        pass

    def eat(self):
        print('son在吃东西');

"""
Son类继承自Father类，Father和Son类中都有eat方法。
Son类里定义了和Father类里同名的函数，调用时会调用Son的方法，而不会调用到Father的方法
"""        
s = Son()
s.eat()

子类扩展父类的方法

在开发中，如果子类的实现包含父类的实现，父类原本的方法实现是子类实现的一部分，此时，我们就可以使用扩展的方式。

  1. 在子类中定义和父类同名的方法，对父类的方法进行重写。
  2. 在需要调用父类实现的位置使用super().父类方法名()来执行父类的方法。
  3. 代码其他的位置针对子类的需求，实现子类功能。
class Father(object):
    def __init__(self):
        pass

    def eat(self):
        print('father在吃东西')

class Son(Father):
    def __init__(self):
        pass

    def eat(self):
        print('son在吃东西');
        super().eat()  # 在Son的eat方法里，需要包含父类的eat功能

s = Son()
s.eat()

调用父类方法的另一种方式

在python2.x版本中，如果想要调用父类的方法，还可以使用父类名.方法(self)的方式调用。

class Father(object):
    def __init__(self):
        pass

    def eat(self):
        print('father在吃东西')

class Son(Father):
    def __init__(self):
        pass

    def eat(self):
        print('son在吃东西');

        #使用父类名.方法名(self) 也可以调用父类的方法
        Father.eat(self)

s = Son()
s.eat()

这种写法在Python3.x版本里依然支持，但是，不建议使用！因为如果父类的类名发生了变化，这里的代码也需要手动的进行修改。

注意：

  • 在开发中，super()和父类名这两种方式不要混合使用，建议使用super(),不要使用父类名的方式。
  • 如果写错了类名，把父类名写成了子类名，会出现递归调用，出现死循环。
class Father(object):
    def __init__(self):
        pass

    def eat(self):
        print('father在吃东西')

class Son(Father):
    def __init__(self):
        pass

    def eat(self):
        print('son在吃东西');
        # Son的eat方法实现里，又调用了 Son的eat方法，会出现递归调用死循环
        Son.eat(self)

s = Son()
s.eat()
