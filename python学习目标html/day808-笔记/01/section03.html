第十六天

线程安全问题

假设两个线程t1和t2都要对全局变量g_num(默认是0)进行加n和减n的操作，最终我们期望的g_num的结果应该依然是0.但是由于是多线程同时操作，有可能出现下面情况：

  1. 在g_num=0时，t1取得g_num=0。在t1做完g_num+=x运算，还未做g_num-=x运算的时候，系统把t1调度为”sleeping”状态，把t2转换为”running”状态，t2读取的g_num已经是t1线程里+=x后的结果，即g_num=1
  2. 然后t2对得到的值进行加1并赋给g_num，使得g_num=2
  3. 然后系统又把t2调度为”sleeping”，把t1转为”running”，线程t1此时在让g_num-=1,得到的g_num值为1而不是0！
  4. 这样导致虽然t1对g_num进行了 += 1 和 -=1 的操作，但是g_num的结果却不是0！

测试1

import threading
g_num = 0

def test(n):
    global g_num
    for x in range(n):
        g_num += x
        g_num -= x
    print(g_num)

if __name__ == '__main__':
    t1 = threading.Thread(target=test, args=(1000,))
    t2 = threading.Thread(target=test, args=(1000,))
    t1.start()
    t2.start()

运行结果：

0
0

测试2

import threading

g_num = 0


def test(n):
    global g_num
    for x in range(n):
       g_num += x
       g_num -= x
    print(g_num)


if __name__ == '__main__':
    t1 = threading.Thread(target=test, args=(1000000,))
    t2 = threading.Thread(target=test, args=(1000000,))
    t1.start()
    t2.start()

运行结果

326130
-641516

结论

  • 如果多个线程同时对同一个全局变量操作，会出现资源竞争问题，从而数据结果会不正确
